## Автоматизация ETL-процессов и создания CRM-инструмента в Google Sheets

Стек технологий: Google Apps Script (JavaScript), Google Sheets, AI (Prompt Engineering).

Роль: Data Analyst / Automation Developer.

**1\. Контекст и бизнес-задача**

Рекламный кабинет Avito предоставляет отчетность в агрегированном виде: одна строка содержит данные по конкретному объявлению и суммарное количество действий (чаты, звонки).

Проблема: Такая структура не позволяет отделу продаж вести индивидуальный учет каждого лида и отслеживать статус работы с конкретным клиентом.

Задача: Разработать инструмент для автоматической трансформации «сырых» данных в атомарный реестр (1 лид = 1 строка) с системой управления статусами.

**2\. Техническое решение (ETL & Automation)**

Решение реализовано с помощью скрипта Google Apps Script, спроектированного в коллаборации с ИИ. Ключевые этапы алгоритма:

1.  Динамическая обработка (Extraction): Скрипт находит нужные столбцы по названиям, что делает систему устойчивой к добавлению новых колонок в выгрузку.
2.  Трансформация (Transformation): \* Unpivoting: Реализован цикл, который «развертывает» одну строку из "Raw" в несколько строк на листе "Result" в зависимости от количества чатов и звонков.
    - Категоризация: Автоматический парсинг названий объявлений для присвоения категорий (например, «DTF-печать», «Визитки»).
    - Дедупликация: Генерация составных ключей (Composite Keys) для предотвращения дубликатов при повторных запусках.
3.  Загрузка и UI (Loading): \* Автоматическое создание выпадающих списков для выбора статуса сделки.
    - Программная настройка цветовой индикации (Conditional Formatting) через код.

**3\. Результаты**

В результате работы был получен автоматизированный инструмент, который заменяет ручной труд аналитика и менеджера по продажам.

![Резултат](assets/final_data.png)

Фрагмент полученной страницы

**Ключевые показатели проекта:**

- Автоматизация: Время на обработку одной выгрузки сокращено с 30-40 минут ручного труда до нескольких секунд выполнения скрипта.
- Масштабируемость: Система готова к обработке любого количества лидов без изменения логики.
- AI Collaboration: Использование ИИ позволило оптимизировать синтаксис скрипта и внедрить сложную проверку уникальности записей в кратчайшие сроки.

**Резюме по использованию ИИ**

Проект выполнен с применением AI-assisted development. Искусственный интеллект выступал в роли эксперта по оптимизации JavaScript-кода и помог в реализации функций программного управления интерфейсом Google Таблиц (валидация данных и условное форматирование). Это позволило сфокусироваться на бизнес-логике трансформации данных, обеспечив при этом техническую чистоту кода.
