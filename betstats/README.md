# **CASE STUDY: Автоматизация риск-мониторинга букмекерской линии**

## **1\. Постановка задачи**

**Контекст:** В букмекерской индустрии данные обновляются ежесекундно. Ошибки в коэффициентах («вилки») ведут к мгновенным убыткам.

Проблема: Аналитики рисков тратят значительное время на ручной поиск аномалий и проверку эффективности прогнозов.

Цель: Разработать инструмент в Google Sheets, который автоматизирует расчет маржи и валидацию исходов.

![Исходные данные](assets/start_data.png)

## **2\. Проектирование решения**

Как архитектор решения, я спроектировал логику, способную обрабатывать разные типы спортивных рынков (двух- и трехисходные) в едином цикле.

Технический стек: Google Sheets + Apps Script (JavaScript).

Методология: Использование AI для генерации кода по четкому техническому заданию (Prompt Engineering) с последующим ручным тестированием.

## **3\. Реализация и AI-оркестрация**

Я поставил задачу нейросети разработать скрипт, который:

1.  Создает кастомное меню в интерфейсе таблиц.
2.  Рассчитывает маржу: $Margin = (\\sum \\frac{1}{Odds_n} - 1) \\times 100$.
3.  Применяет условное форматирование к строкам с отрицательной маржей.

![Часть кода](assets/part_code.png)

## **4\. Результаты работы скрипта**

После выполнения кода система автоматически разметила данные, выделив критические зоны для риск-менеджера.

![Резултат](assets/final_data.png)

**Ключевые показатели:**

- Скорость: Обработка 1000+ строк занимает менее 3 секунд.
- Точность: Автоматическая идентификация «вилок» (подсветка красным при марже < 0%).
- Аналитика: Статус «Correct/Incorrect» позволяет мгновенно оценить проходимость прогнозов.

## **5\. Финальная проверка**

Я провел финальный аудит работы алгоритма. Скрипт корректно выявил аномальные коэффициенты (строки с маржей до -55%), которые возникли из-за ошибок в исходном датасете. Это подтверждает готовность инструмента к работе в реальных условиях для предотвращения финансовых потерь.
